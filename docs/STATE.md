## Decisions
- DECISION: scope "main platform" to the surfaces CLAUDE.md itself names as platform chrome (`/`, `/login`, `/host`, `/admin`) — confirmed by reading Landing.tsx, Login.tsx, Signup.tsx, HostLayout.tsx, AdminLayout.tsx, admin/Overview.tsx: all use only semantic Tailwind tokens (app-bg, text-foil-static, bg-foil, glow-accent) except 3 `text-noir-900` CTA-button-text instances, now swapped to `text-white` for contrast against the new blue/violet `.bg-foil` gradient.
- DECISION: leave BUILTIN_BORDERS (src/lib/borders.ts), EVENT_TEMPLATES themeVars, GuestNav, and all per-event guest/admin product surfaces untouched — customer-facing per-event theming, not "main platform" branding; touching them risks regressing live customer events.
- DECISION (pivoted mid-task per user input): for the background "particle streams," use a raw WebGL1/GLSL fullscreen-shader (src/components/ui/SpectrumField.tsx) instead of CSS keyframes or Three.js/R3F/GSAP — the codebase already has a hand-rolled WebGL1 shader pipeline for the booth (src/lib/shaders.ts, e.g. `aurora-lumina`'s wave-band technique, `champagne-sparkle`'s twinkle points) using the exact same fullscreen-triangle/hash-noise/screen-blend conventions, just gold-tinted; SpectrumField generalizes that technique to the platform's 7-hue palette with no texture input (pure generative). No new dependency needed (R3F/Three would be a first-time use outside the booth; GSAP isn't an existing runtime dep — only in the standalone hyperframes export). Removed the CSS `beam-in`/`stream-drift` keyframes added in Step 1 once this was decided (dead code).
- DECISION: the frame gallery cards themselves (FrameShowcase.tsx) stay DOM + `motion/react` (already the established idiom for entrance choreography, e.g. BeamIn.tsx, Welcome.tsx) — real accessible content (images, labels), not baked into a texture. WebGL is reserved for the generative background only.
- DECISION: 7-hue "beam spectrum" palette lives as local constants in FrameShowcase.tsx + duplicated inline in SpectrumField.tsx's GLSL (not a shared TS/GLSL module) — two small, self-contained consumers, not worth a shared palette module for this scope.
- DECISION: two Higgsfield-generated images (AR-glasses portrait for "Booth", beam-wall venue shot for "Wall") are wired in as remote CDN URLs (`https://d8j0ntlcm91z4.cloudfront.net/...`) rather than downloaded into src/assets/landing/, because this sandbox's egress proxy blocked that CDN host (403, confirmed via `curl $HTTPS_PROXY/__agentproxy/status`) — per the proxy's own guidance ("do not retry or route around it"), did not attempt workarounds. FrameCard has an `onError` fallback to the icon/gradient look if the image fails to load (verified via screenshot — this sandbox's Playwright browser also can't reach that CDN, so it correctly falls back). Flagged to the user; a follow-up should swap these for self-hosted files once the bytes can be gotten into the repo.
