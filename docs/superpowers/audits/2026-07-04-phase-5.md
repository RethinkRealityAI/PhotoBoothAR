## Adversarial audit (3-dimension review workflow + verify pass)

12 agents (3 reviewers √ó high effort, then per-finding verification). 9 raw
findings, **8 confirmed**, 1 uncertain. Dispositions:

### FIXED
1. **HIGH ‚Äî cross-tenant leak of `contribute_token` + `recipient_email`.**
   `cards_public_read` had no role clause, so it applied to `authenticated` and
   OR-combined with the member policy; the full authenticated column grant then
   let ANY logged-in user (incl. other-tenant members / open signups) read every
   published/rendered card's secret token + recipient PII across all orgs.
   **Fix (migration 008, APPLIED LIVE):** scoped `cards_public_read` to the
   `anon` role only. Members read own cards (all columns) via the member policy;
   anon reads only the column-granted safe fields of published cards; authed
   non-members lose the cross-tenant path. **Verified live ‚Äî 5/5 probes:** authed
   stranger ‚Üí 0 rows; anon `contribute_token`/`recipient_email` ‚Üí denied; anon
   safe column of published card ‚Üí allowed; anon collecting card ‚Üí 0 rows.
2. **HIGH ‚Äî guest quota bypass via rotating client-chosen `sessionId`.**
   Per-(card,session) 10/hr cap is defeated by a new sessionId per request.
   **Fix:** added a hard per-card ceiling `CARD_MAX_CONTRIBUTIONS = 500` ‚Üí 429
   `card_full` in both init and finalize (media + text paths). Bounds worst-case
   spam/storage cost regardless of sessionId rotation.
3. **MEDIUM ‚Äî unpublishing a pinned landing card stranded `/e/:slug` guests.**
   **Fix:** `doUnpublish` now clears the pin (`updateEventConfig(..., {
   primary_card: null })` + `refreshConfig()`) when the unpublished card is the
   landing; `EventIndexRedirect` falls through to `landingRoute` when the pin is
   null (confirmed + commented).
4. **LOW ‚Äî `card-publish` email link trusted the request `Origin` header and
   interpolated the URL unescaped.** **Fix:** link base is now `PUBLIC_SITE_URL`
   env only (hardcoded `https://beamwall.app` fallback; Origin removed as a
   source); the URL is `escapeHtml`'d at both href + visible-text spots.
5. **(UNCERTAIN) publish/unpublish clobbering a `rendered` card / `published_at`.**
   **Fix:** publish stamps `published_at` only from `collecting` (no clobber on
   re-publish); unpublish ‚Üí `collecting` leaves `card_renders` intact.

### ACCEPTED (documented in-code)
