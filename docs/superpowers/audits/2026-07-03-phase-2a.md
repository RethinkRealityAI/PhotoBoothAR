## Audit findings

- Line review of both functions: input validation solid (uuid/regex checks,
  op allowlist, object-shape checks); every post/settings query double-scoped
  (`id` + `event_id`); token never stored raw anywhere (client generates,
  hashes, discards).
- Grep audit: remaining `/admin` literals only in `AdminGate.tsx` (the legacy
  layout, intentionally untouched) and the `useStudioBase()` default.
- Accepted quirks (in-code comments): archived events show the "ended" screen
  instead of the studio; event theme lingers on returning to `/host`; runtime
  `/e/:slug/admin` now falls through to the event redirect (studio moved).
- Known limitation carried from Phase 1: the container's network policy blocks
  `*.supabase.co`, so the fresh-account browser E2E (signup → wizard → capture
  → wall) cannot be driven from this environment. Structural verification +
  live SQL/RLS probes stand in; the first real E2E runs on the deployed
  beamwall site once created (user checklist).
- Manager token threat model: token grants moderation + wall settings only;
  quota/billing/catalog untouched by design.
