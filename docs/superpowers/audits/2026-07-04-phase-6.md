# Phase 6 Audit — Premium MP4 Keepsake Film (HeyGen HyperFrames)

**Date:** 2026-07-04 · **Branch:** `claude/photo-wall-saas-strategy-xd1rxf`

## Backend decision

**HeyGen HyperFrames** chosen over Remotion (see plan DECISION LOG): Apache-2.0
free at any team size (Remotion is $100/mo min for 4+ person for-profits), no
AWS (HeyGen cloud API or self-host), and a keepsake film is a bespoke cinematic
composition anyway — so Remotion's React-template-reuse edge is moot here. The
render stays behind a pluggable `RENDER_BACKEND` env, so this is reversible.

## Delivered

- **HyperFrames composition** `hyperframes/keepsake-film/` (index.html + vendored
  GSAP + build-render.mjs + fixture): data-driven 1080p film — title card →
  per-contribution scene (photo Ken-Burns + caption / typographic text card /
  video) → closing card, themed by event colors. Guest text inserted via
  `textContent` only (no markup injection). Kept out of the app vite/tsconfig
  build.
- **`card-render` + `card-render-status`** edge functions (authored; deployed
  this pass). Deluxe-only entitlement (per-event; NO subscription fallback,
  unlike card-publish), published-card gate, 30-credit spend-first with refund
  on every non-started path.
- **Client:** `renderCard`/`fetchRenderStatus`/`renderDownloadUrl` in
  `src/lib/cards.ts`; "Keepsake film" section in CardsTab (guarded by
  `cardsPremiumRender` + published) with queued/rendering/done + signed download
  or the `render_not_configured` upsell.

## Local render — SUCCEEDED

23.0s H.264 1080p/30fps MP4 (6 MB) rendered fully in-container from the fixture
(title + photo Ken-Burns + text card + closing), verified frame-by-frame.
Container-specific toolchain fixes (CLI `--ignore-scripts`, an ffmpeg shim for
the libx264 build, Chromium headless shell) are isolated in `.hf-bin/` +
gitignored.

## Gate

tsc clean · 50/50 vitest · vite build OK · `VITE_EVENT=hope-gala` smoke OK · no
`hyperframes` chunk in the app bundle (composition fully isolated).

## Audit findings & dispositions

1. **Credit safety — PASS.** Spend-first (`spend_credits`), refund on every
   non-started outcome: disabled backend, insert failure, submit failure,
   render failure. Reviewed both functions line-by-line.
2. **Concurrent-poll double-refund — GUARDED (verified).** Both terminal
   transitions in card-render-status use `.eq('status','rendering')` conditional
   updates, so only the winning poll refunds/finalizes exactly once. Transient
   poll errors keep the row `rendering` (no refund).
3. **Entitlement — PASS.** Deluxe event tier or legacy slug only, checked
   server-side; a Pro subscription deliberately does NOT unlock the deluxe film
   (per-event package), matching `ENTITLEMENTS.deluxe.cardsPremiumRender`.
4. **Ships safe by default.** `RENDER_BACKEND` unset → 503 `render_not_configured`
   + full refund; no guest media leaves our infra and no credits are consumed
   until a HeyGen key + `RENDER_BACKEND=hyperframes` are set. Blast radius until
   go-live is effectively zero.
5. **Private renders bucket:** card-render-status signs a 1h MP4 download URL
   with the service role (the `renders` bucket has no member-read policy and
   migrations for it are additive-only); membership is verified before signing.

## KNOWN LIMITATIONS (must validate before charging for the film)

1. **HyperFrames Cloud Render API contract is ASSUMED (flagged in-code).** HeyGen
   publishes no openly-fetchable REST reference (developers.heygen.com 403s the
   container; the GH repo documents only local CLI + AWS-Lambda self-host). The
   submit/poll endpoints + field names are modelled on HeyGen's `x-api-key`
   convention + the public HyperFrames MCP render tools and are isolated to ONE
   place in each function. **Confirm against the real API before go-live**; the
   credit/refund/reconcile plumbing is contract-independent. Documented fallback:
   the AWS-Lambda self-host path (`npx hyperframes lambda`) or the Cloudflare
   template.
2. **Video contributions don't composite in the LOCAL producer** (runtime
   `<video>` renders black; frame extraction maps to elements present at the
   initial DOM scan). The scene degrades gracefully to the themed backdrop +
   caption. Photo + text are fully verified. Validate real video playback on the
   production cloud render, or switch the video scene to a poster-still
   treatment. The interactive WEB card (standard tier) plays videos fine — this
   limitation is only the premium MP4.

## Go-live actions (added to the consolidated checklist)

Upload `hyperframes/keepsake-film/` to HeyGen once → record its asset id.
Secrets: `RENDER_BACKEND=hyperframes`, `HEYGEN_HYPERFRAMES_API_KEY`,
`HEYGEN_HYPERFRAMES_ASSET_ID`, optional `HEYGEN_HYPERFRAMES_API_URL`. Leave
`RENDER_BACKEND` unset to keep the film safely disabled.
