## Audit findings & dispositions

1. Webhook signature scheme line-reviewed — correct raw-body HMAC, multiple
   v1 support, timestamp tolerance, constant-time compare. PASS.
2. **Partial-fulfilment retry risk (accepted, documented):** the claim is
   released on failure so Stripe retries; if a crash lands between the
   `event_plans` insert and the credit grant, a retry re-runs the whole
   handler → a duplicate `event_plans` history row and a double credit grant
   are possible in that narrow window. Mitigation planned: unique ref on
   ledger grants (make `grant_credits` idempotent per stripe_event) when
   billing goes live with real keys.
3. **Anonymous-guest Pro detection (documented deviation):** RLS hides
   `subscriptions` from guests, so a Pro org's *free* event still watermarks
   for guests. Fix scheduled in Phase 4: denormalize the pro floor onto
   `events.plan_tier` in the webhook.
4. Server-side maxPosts cap not yet enforced in `submit-post` (frozen this
   phase); free-tier abuse remains bounded by the 30/hr quota. Scheduled for
   Phase 4 alongside the AI work.
5. No client can write billing tables (RLS member-SELECT only) and
   `grant_credits`/`spend_credits` are service-role-only — verified by the
   Phase 1 probe suite which remains green.
